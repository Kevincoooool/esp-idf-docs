## 一、拿到板子后先做什么

### 1.0 获取例程（必读）

1. 淘宝 **酷世DIY** 购买 P4C5 开发板  
2. **下单后联系客服**，索取例程资料  
3. 客服通过 **百度网盘** 发送压缩包（含 `P4_C5_4.3_Firmware/` 及说明）  
4. 解压到**无中文路径**（如 `D:\Firmware\`）

### 1.1 开箱检查

| 序号 | 物品 | 检查项 |
|------|------|--------|
| 1 | ESP32P4-C5 4.3寸触摸屏开发板 | 屏幕无裂痕、排针无弯折 |
| 2 | USB Type-C 数据线 | 必须是**数据线**（非纯充电线） |
| 3 | TF 卡（视套餐） | 多媒体例程需要，FAT32 格式 |
| 4 | 例程资料 | 淘宝下单后 **联系客服**，**百度网盘** 发链接（非公开下载） |

### 1.2 上电验板（无需编程）

若出厂已烧录 `p4c5_board_test` 固件：

1. USB 连接板子 Type-C 与电脑
2. 等待约 3 秒，屏幕自动亮起
3. 出现 **NODE P4 赛博风格界面** 表示正常

### 1.3 第一次自己编译烧录

```powershell
# 百度网盘下载解压后（路径不要含中文）
cd D:\Firmware\P4_C5_4.3_Firmware\04.advanced.p4c5_board_test

idf                          # Windows PowerShell 导入 IDF 环境
idf.py set-target esp32p4   # 首次必须执行
idf.py build
idf.py -p COM15 flash monitor   # COM 口改成你的实际端口
```

**成功标志**：编译无 ERROR；烧录 100%；串口有 `KSDIY_TEST` 等日志；屏幕显示 NODE P4 界面。

### 1.4 板级测试功能导航

| 入口 | 功能 |
|------|------|
| 主界面 | 硬件健康度、频谱动画 |
| 硬件检测 | AXP / ES8311 / ES7210 / GPIO 按键 / I2C |
| Wi-Fi 扫描 | C5 扫描周边 AP |
| MIPI CSI | SC2336 相机预览（需安装传感器） |
| USB 相机 | UVC 预览（需 USB Host 接相机） |
| IMU | LSM6DS3 加速度/角速度 |
| 语音 | ES7210 + 唤醒词演示 |

逐项进入，**绿色/OK** 表示该项硬件正常。

## 三、为什么每个例程都能在屏上显示？

即使 `01.basic.gpio` 也会在 4.3 寸屏显示状态，因为所有例程共享 **`ksdiy_p4c5_bsp`** 组件：

```c
nvs_flash_init();
ksdiy_lvgl_port_init();   // 内部：AXP 上电 → MIPI DSI → 触摸 → LVGL 9
```

之后用 LVGL 标准 API 画界面，操作前加锁：

```c
if (ksdiy_lvgl_lock(100)) {
    lv_obj_t *label = lv_label_create(lv_scr_act());
    lv_label_set_text(label, "Hello P4C5");
    lv_obj_center(label);
    ksdiy_lvgl_unlock();
}
```

在自己的空白工程里也可通过在线组件使用，见 [P4C5在线组件使用说明](./P4C5在线组件使用说明.md)。

## 五、软件架构（进阶必读）

### 5.1 显示栈

```text
你的 LVGL UI
    ↓ ksdiy_lvgl_lock / unlock
esp_lvgl_adapter（LVGL 9）
    ↓
esp_lcd ST7102 MIPI 驱动
    ↓
MIPI DSI → 4.3 寸屏
```

触摸：ST7123 → LVGL indev

### 5.2 音频栈

```text
WAV/PCM/AVI 音频轨 → audio_decoder / I2S → ES8311/ES7210 → 喇叭/麦克风
```

### 5.3 Wi-Fi 栈

```text
你的 app → esp_wifi_* API → esp_wifi_remote → esp_hosted SDIO → ESP32-C5
```

### 5.4 视频解码栈

```text
TF 卡 AVI → avi_parser → P4 硬件 JPEG + PPA 缩放 → esp_lcd / LVGL
```

## 六、日常开发工作流

```powershell
cd P4_C5_4.3_Firmware\03.development.lvgl_v9
idf.py build flash monitor
```

| 命令 | 用途 |
|------|------|
| `idf.py menuconfig` | 改 COM 口、分区、ESP-Hosted 等 |
| `idf.py fullclean` | 清 build 后重编 |
| `idf.py build app-flash` | 只烧 app 分区 |

**读 log**：`I`=正常，`W`=警告，`E`=错误，`panic`=严重错误需复制完整 backtrace。

⚠️ 勿运行仓库中的 `build kill.bat`（会删 build/.clangd/.vscode）；清理 clangd 索引用 `clangd_kill.bat`。

## 七、验板失败排查

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 找不到 COM 口 | 纯充电线、驱动未装 | 换数据线，装 USB 串口驱动 |
| 烧录失败 | 端口占用 | 关闭其他串口工具 |
| 屏幕不亮 | 未烧录成功 | 重新 flash，看 AXP 初始化 log |
| Wi-Fi 扫描为空 | C5 未就绪 | 等 10～20 秒，查 esp_hosted log |
| 音频报错 | Codec 未检测到 | 硬件检测页对比 I2C 设备 |

## 八、常见问题 FAQ

### Q1：屏幕亮了但触摸没反应？

确认使用 `ksdiy_lvgl_port_init()`；查 ST7123 初始化 log；跑板级测试硬件检测页。

### Q2：Wi-Fi 连不上？

等 C5 初始化（`Transport active`）；改 SSID/密码；先用 `05.test.wifi_scan_c5`。

### Q3：TF 卡读不到？

FAT32 格式；先跑 `02.beginner.sdmmc_sd`。

### Q4：AVI 无画面或卡顿？

确认 **MJPEG 编码 AVI**；文件放 `/sdcard/videos/`；分辨率建议 ≤ 800×1280。

### Q5：编译找不到组件？

```powershell
idf.py fullclean
idf.py set-target esp32p4
idf.py build
```

需联网下载 `managed_components/`。

### Q6：可以用 Arduino IDE 吗？

本例程基于 ESP-IDF，暂不支持 Arduino IDE。
